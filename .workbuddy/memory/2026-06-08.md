# 2026-06-08

## VSCode 扩展开发

### 初始创建
- 在 `vscode-extension/` 下创建了完整的 VSCode 扩展项目
- 源码：`src/extension.ts`, `src/xml2UdlEngine.ts`, `src/treeDataProvider.ts`, `src/udlDocumentProvider.ts`
- 移植了 demo/xml2Udl.js 的全部解析逻辑到 TypeScript，使用 @xmldom/xmldom 代替浏览器 DOMParser

### 架构决策
- 使用 TreeView（侧边栏）展示解析后的文件列表，按类型分组
- 使用虚拟文档 URI scheme `xml2udl-udl://` 显示 UDL 内容（不产生磁盘文件）
- 点击 TreeView 项 → 在右侧编辑器列打开虚拟文档，利用 VSCode 原生语法高亮

### Bug 修复
1. **Node 版本兼容**：系统 Node 11 不支持 TypeScript 5.x 的 `??` 语法 → tasks.json 和 package.json 脚本改用 managed Node 22.22.2 显式路径
2. **TreeView 不可见**：解析后侧边栏不会自动切换到 XML to UDL 面板 → 添加 `vscode.commands.executeCommand('xml2udl.treeView.focus')`，并用 `treeView.message` 替代弹窗通知；类型分组默认展开 (Expanded)
3. **Class 解析丢失关键字**：`generateUdlFromXmlElement` 将 XML 包了一层 `<root>` 但传递给了 xml2Udl 的是 root 元素而不是其中的 Class 元素，导致所有 class 解析逻辑被跳过（走到 unknown2Udl 只返回 textContent）。修复：直接用 `doc.documentElement` 作为根元素传给 xml2Udl
